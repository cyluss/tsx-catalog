# Tabs

<dl>
  <dt>원칙</dt>
  <dd>같은 수준의 콘텐츠를 공간 효율적으로 전환한다. 탭 간 콘텐츠는 독립적이어야 하며, 순서에 의존하는 흐름에는 사용하지 않는다.</dd>
  <dt>사용 시점</dt>
  <dd>
    <div class="do-dont">
      <div class="do"><strong>써야 할 때</strong><p>설정 페이지 (일반·보안·알림)</p><p>대시보드의 뷰 전환 (일별·주별·월별)</p><p>상세 페이지의 섹션 분리 (정보·리뷰·FAQ)</p></div>
      <div class="dont"><strong>쓰지 말아야 할 때</strong><p>순서가 있는 단계 — Stepper 사용</p><p>탭이 2개 이하 — 그냥 표시</p><p>탭 안에 Dropdown — 접근성 문제</p></div>
    </div>
  </dd>
  <dt>선택 규칙</dt>
  <dd>탭 수는 2–5개가 적당하다. 탭 레이블은 짧게(1–2단어), 아이콘만 단독 사용은 금지한다.</dd>
</dl>
